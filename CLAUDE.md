# PayPayAnalyzer - CLAUDE.md

## プロジェクト概要

PayPayの利用履歴CSVを読み込み、指定期間内の支払い出金を取引先ごとに集計して
出金ランキングTOP30を一覧表示するWebアプリ。

- **技術スタック**: React + Vite + JavaScript + Papa Parse
- **動作**: ブラウザ内のみで処理（サーバー・DB不要）

## 主要な実装メモ

### CSVパース（Papa Parse）
- `header: false` で生配列取得 → 先頭15行を走査してヘッダー行を自動検出
- PayPayのCSVは先頭にメタ行が入る場合があるため

### 対応する列名（柔軟な検出）
| フィールド | 対応列名 |
|---|---|
| 取引日 | 取引日 / 取引日時 / 日付 / date |
| 出金金額 | 出金金額 / 支払い金額 / 出金金額（円）など |
| 取引先 | 取引先 / 店舗名 / 支払先 / vendor / store |
| 取引内容 | 取引種別 / 取引内容 / 種別 / type |

列名は完全一致のほか「含む」でもマッチする（例: "出金金額（円）" → "出金金額" に一致）。

### 日付パース上の注意
PayPayのCSVは `2026/3/31 19:28` のように月・日がゼロ埋めなし。
`new Date('2026-3-31T00:00:00')` はブラウザによってInvalid Dateになるため、
`padStart(2, '0')` でゼロ埋めしてから `new Date('2026-03-31T00:00:00')` でパースしている。

### 支払いフィルター
取引内容列が検出された場合、「支払」「払い」「決済」のいずれかを含む行のみ集計対象。
チャージ・送金・受け取りなどを除外する目的。

### 金額パース
カンマ・円記号を除去し `Math.abs()` で絶対値を取る（マイナス表記にも対応）。

## Git運用ルール

### 基本方針
- コードを変更するたびに、変更内容をコミットしてGitHubにプッシュすること。
- コミットメッセージは日本語でも英語でも可。変更内容を簡潔に記述すること。
- `main` ブランチを常に最新の状態に保つこと。

### リモートリポジトリ
- URL: https://github.com/yukiyuki-hub/PayPayAnalyzer.git
- デフォルトブランチ: `main`

### コミット・プッシュの手順
1. 変更内容を確認: `git status` / `git diff`
2. ステージング: `git add <ファイル名>` (必要なファイルのみ)
3. コミット: `git commit -m "変更内容の説明"`
4. プッシュ: `git push origin main`

### 注意事項
- `.env` ファイルや認証情報・APIキーなどの機密情報はコミットしないこと。
- `git add -A` や `git add .` は機密ファイルを誤って含める可能性があるため、ファイルを個別に指定すること。
- 破壊的な操作 (`git reset --hard`, `git push --force` など) はユーザーの明示的な指示がある場合のみ実行すること。

## 開発ガイドライン

- 初心者が読むことを想定し、主要な処理にはコメントを入れること。
